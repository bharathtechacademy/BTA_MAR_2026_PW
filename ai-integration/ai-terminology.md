# AI - Key Terminology
When it comes to AI tools available in the current market, there are multiple things:
- Co-pilots
- Coding Assistants
- Conversational AI tools
- LLMs
- RAG
- MCP
- etc.

## Conversational AI tools
Set of web applications or desktop applications integrated with an AI model that is going to allow us to chat with this particular tool and respond with a single answer like a human. 

Example : Chart GPT, Gemini, Claude, etc. 

How will a conversational AI tool work in the background? 

User adds the input in the chart box. => System processes the input and shares it with LLM (Large Language Model) => LLM thinks about the problem statement. => LLM respond back with a single answer. 

Explain encapsulation in Java => "Explain", "encapsulation", "in", "Java" => [1021,0.001 ,5678,0.32 , 6543,0.003, 5467, 0.25] => Match with the trained content stored in the LLM => Encapsulation is all about binding the data and methods together. 

## Copilots

Copilot is an AI assistant and a plugin installed inside the IDE. Especially, it is going to auto-complete the task, and also it is going to provide you the code option within the IDE itself. 

Example : GitHub Copilot , Claude , Tab nine

Co-pilot can write the code for the new functionality based on the existing code. 
Co-pilot can explain the existing code written by another person in your framework. 
Copilot can debug the issue whenever there is any error identified on your project. 
Co-pilot can perform unit testing based on the existing code. 
Co-pilot can refactor the code, and it can correct the format. 
Co-pilot can help us to learn the new concepts introduced in the programming language and text tag. 

## Coding assistants

Coding Assistant is an IDE integrated with AI models by default. 

Examples : Cursor, Wind Surf, Claude Code, Anti-Gravity , Codex etc..

## LLM ?

LLM is a large language model that can understand your language and generate the text, write the code, and answer generic questions. 

Examples :
GPT 5.5 
Gemini Pro 3.1 
Claude Opus 

How is an LLM trained internally? 

LLM will be trained in three different stages. 
1. Pre-training stage. => Where the model is going to reads huge data from the books, websites, code, and articles available on the internet 
2. Fine Tuning Stage => Now we are going to add additional instructions to provide us the accurate answer to be specific to the particular domain. 
3.Reinforcement Learning with Human Feedback  Stage => LLM will provide the answer, and when the user is going to give the feedback about the answer, it is going to collect that particular feedback, remember it, and improve the next answer. 


## RAG (Retrieval Augmented Generation )
RAG is a framework. Whenever a user is going to ask any question, the RAG framework is going to help us to identify the relevant data only from the trained content inside the LLM. Later, it is going to share the details to the LLM about particular information, and the LLM is going to generate the answer. 0

## MCP (Model Context Protocol )
MCP can take the knowledge from the AI model available in the background, and it can take the instructions from the user. Finally, it can take control of your computer and software to perform different types of tasks.
