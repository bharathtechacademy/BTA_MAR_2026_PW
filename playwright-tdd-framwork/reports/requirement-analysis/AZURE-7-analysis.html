1. User Story Objective & Detailed Interpretation

Original Story (As-a / I-want / So-that)

As a user, while logging into the application, I should see a Cookies Consent banner at the bottom of the login page with necessary details and interactive controls.

Interpretation

The story requires the application to display a GDPR/privacy-compliant cookie consent popup on the Creatio CRM login page (https://accounts.creatio.com/login/alm). The banner must appear automatically when the page loads, present the consent message, offer granular control via toggle switches (Necessary, Preferences, Statistics, Marketing), allow the user to expand detail via a "Show details" hyperlink, and dismiss the banner through one of three action buttons: Allow all, Allow selection, or Deny.

Key Behavioural Rules

  • The Necessary toggle must be permanently disabled (always-on) — users cannot opt out of essential cookies.
  • Preferences, Statistics, and Marketing toggles are user-configurable.
  • Clicking any of the three action buttons (Allow all / Allow selection / Deny) must close the consent popup.
  • Allow all enables all toggles; Deny disables all optional toggles; Allow selection respects current toggle states.
  • The "Show details" hyperlink expands an additional details/about section within the banner.
  • The consent message text must match verbatim the content defined in the acceptance criteria.

2. What We Are Trying to Achieve

  • Regulatory compliance: Ensure the application meets GDPR/CCPA cookie consent requirements before users interact with any part of the application.
  • User transparency: Give users clear, plain-language information about what cookies are collected and why.
  • Granular user control: Allow users to selectively enable/disable non-essential cookie categories.
  • UX integrity: The consent banner must not interfere with the primary login form — it should be positioned below the form and not block login actions.
  • Persistence: Verify that the user's cookie preference is respected and not re-prompted unnecessarily on subsequent visits (TBD).
  • Test automation readiness: The story has Automation Required = True, so page objects and automated tests must be built as part of delivery.
