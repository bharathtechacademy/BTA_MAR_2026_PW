# Day-to-day tasks related to software testing 

1. Requirement Analysis for the New User Story 
2. Test design for the new user story. 
3. Manual test execution of each and every test case, identifying the defects and logging the defects in JIRA or ADO 
4. End of test report after completing the test execution and the analysis of overall execution flow. 
5. Automation feasibility analysis for the new test cases. 
6. Automating the new manual test cases by following the existing framework 
7. Maintaining the automation test cases and fixing the errors in the existing automation test scripts. 
8. Review the PR when someone is going to write all the test cases by following all the standards followed by my organization. 
9. Create New Automation Framework
